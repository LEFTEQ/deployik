## `DeriveAppProfile` Logic

1. Check `appPkg.Dependencies` + `appPkg.DevDependencies` for:
   - `next` â†’ `framework="nextjs"`, `outputDirectory=".next"`
   - `vite` â†’ `framework="vite"`, `outputDirectory="dist"` (override from vite config if present)
   - `astro` â†’ `framework="astro"`, `outputDirectory="dist"`
   - otherwise â†’ `framework="static"`, `outputDirectory="dist"`
2. `Buildable = appPkg.Scripts["build"] != ""`
3. `SuggestedBuildCommand`:
   - Turborepo detected + app has `Name` â†’ `{pm} turbo run build --filter={app.Name}`
   - Nx detected â†’ `{pm} nx build {app.Name}`
   - Otherwise â†’ `{pm} run build` (using the resolved package manager)

## `ExpandGlobs` Algorithm

Globs match against the flat tree (path list from GitHub). For each glob:
- Exact path: single match
- Trailing `/*`: match one directory level
- `/**`: recursive match

Output is sorted and deduped. Only paths that correspond to actual directories in the tree (i.e. some path in tree starts with `{appPath}/`) are included â€” avoids returning phantom app paths.

## Testing Pattern

`inspect_test.go` uses a `fakeRepoInspector` struct implementing `RepoInspector`. Tests never hit GitHub:

```go
fake := &fakeRepoInspector{
    tree: []string{"packages/web/package.json", "packages/api/package.json", "pnpm-workspace.yaml", ...},
    files: map[string][]byte{
        "pnpm-workspace.yaml": []byte("packages:\n  - packages/*"),
        "packages/web/package.json": []byte(`{"name":"@scope/web","scripts":{"build":"vite build"},...}`),
    },
}
report, err := monorepo.Inspect(ctx, fake, "owner", "repo", "main")
```

`InspectHandler` exposes `InspectFn func(...)` field â€” set it to a stub in handler tests.

## Edge Cases

- **Malformed `pnpm-workspace.yaml`**: `ParsePnpmWorkspaceYAML` returns error â†’ `Inspect` falls back to `singleAppReport`. Graceful.
- **App `package.json` absent**: `inspectApp` returns zero-value `App{}`. Filtered out in `Inspect` by checking `app.Name == "" && app.Path == ""` (for monorepo paths, path is always non-empty, so filter by `app.Name != "" || app.Path != ""`).
- **Truncated tree**: `report.Truncated = true`. The `PickApp` UI should warn the user that some apps may be missing.
- **No build script**: `App.Buildable = false`. Frontend should warn; deploying a non-buildable app will produce an empty build output.
- **Single app detected as monorepo workspace**: unlikely but possible if `package.json` lists `workspaces: ["./"]`. `ExpandGlobs` would yield the root path â†’ single-app monorepo. Behaves correctly since `App{Path: ""}` is filtered and replaced.
