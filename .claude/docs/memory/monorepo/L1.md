# Monorepo - Patterns & Rules
> Load when touching monorepo detection, the inspect endpoint, or the new-project pick-app flow.
> Last validated: 2026-04-19

| File | Purpose |

## Core Types

```go
type Report struct {
    IsMonorepo     bool      `json:"is_monorepo"`
    PackageManager string    `json:"package_manager"` // "auto" | "bun" | "pnpm" | "npm" | "yarn"
    Tooling        []Tooling `json:"tooling"`          // []Tooling{} when none
    Apps           []App     `json:"apps"`             // always >= 1; single entry (Path="") for non-monorepo
    Truncated      bool      `json:"truncated"`        // GitHub tree API truncation signal
}

type App struct {
    Name                  string `json:"name"`
    Path                  string `json:"path"`                    // "" for non-monorepo root
    Framework             string `json:"framework"`               // "nextjs"|"vite"|"astro"|"static"
    OutputDirectory       string `json:"output_directory"`
    SuggestedBuildCommand string `json:"suggested_build_command"` // turbo/nx-aware
    Buildable             bool   `json:"buildable"`
}
```

## Detection Logic (pure, no IO)

Workspace detection priority:
1. `pnpm-workspace.yaml` globs
2. Root `package.json` `workspaces` (npm/yarn/bun all use the same field)
3. If no globs found → single-app report

Tooling detection (`DetectTooling`): looks for `turbo.json` → `ToolingTurborepo`; `nx.json` → `ToolingNx` in root files.

Package manager detection (`DetectPackageManager`): `bun.lockb`/`bun.lock` → bun; `pnpm-lock.yaml` → pnpm; `yarn.lock` → yarn; `package-lock.json` → npm; fallback → `"auto"`.

`DeriveAppProfile` maps `next`, `vite`, `astro` deps → framework; reads vite `build.outDir` for output override.

`SuggestedBuildCommand` is turbo/nx-aware: `pnpm turbo run build --filter=@scope/app` (when Turborepo detected) or plain `pnpm run build` per package manager.

## Inspect Orchestration

`Inspect(ctx, RepoInspector, owner, repo, ref)` pipeline:
1. `GetTree` once (flat path list + truncated flag)
2. Build `rootFiles` set (top-level paths, no `/`)
3. Detect package manager + tooling from `rootFiles`
4. Fetch `package.json` + `pnpm-workspace.yaml` (pre-checked against `rootFiles` to avoid phantom HTTP round-trips)
5. Compute + expand workspace globs via `ExpandGlobs`
6. Non-monorepo path: return single-app report
7. Per-app fetches via `errgroup` capped at 5 concurrent: `{appPath}/package.json` + optional vite config (only when vite dep present AND file in tree)

## `RepoInspector` Interface (mockable seam)

```go
type RepoInspector interface {
    GetTree(ctx, owner, repo, ref) (paths []string, truncated bool, err error)
    GetFileContent(ctx, owner, repo, ref, filePath) ([]byte, error)
}
```

`ErrFileNotFound` is the sentinel returned when a file is absent. `Inspect` treats it as soft (skips that file gracefully).

The `gitHubInspectorAdapter` in `handlers/inspect.go` translates `gh.ErrNotFound` → `monorepo.ErrFileNotFound` so the two packages remain dependency-free of each other.

## Do / Don't

| Do | Don't | Why |
|----|-------|-----|
| Use `ErrFileNotFound` to signal absent files | Return a nil slice or a generic error | `Inspect` uses `errors.Is` to distinguish absence from transient failure |
| Keep detection helpers in `detect.go` (pure, no IO) | Put IO inside detection functions | Testability — `inspect_test.go` uses a fake `RepoInspector` |
| Set `InspectFn` field on `InspectHandler` in tests | Mock the GitHub client | Avoids HTTP-level mocking; tests inject a `*monorepo.Report` directly |
| Return a fixed error string to the client on failure | Surface internal error details | Security — internal errors logged server-side only |

## Gotchas

- **Single-app repos still go through inspect** — `Report.IsMonorepo == false` with one `App{Path: ""}`. The frontend skips Step B but still uses the auto-fill path via `useEffect`.
- **`Truncated=true`** means GitHub returned a partial tree. Per-app profile may be incomplete. The frontend surfaces a warning via `report.truncated`.
- **`apiBase` in `github.Client` is a `var`** (not `const`) to allow `SetTestAPIBase(string) func()` override in tests.
- **`GetTree` context-carries** — `GetFileContent` and older GitHub client methods still don't accept context. The incremental migration is intentional.
- **`inspectLimiter` is 20/min/IP** — separate from `mutationLimiter`. High-frequency branch-switching in the new-project UI is expected; the limit is permissive but prevents abuse.
