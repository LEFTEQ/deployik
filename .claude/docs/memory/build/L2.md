# Build Pipeline - Deep Reference
> Load for pipeline bugs, Dockerfile regressions, adding a new framework runtime, or blue-green invariants.
> Last validated: 2026-04-16

## Architecture

```
DeploymentHandler.Trigger
  └── creates db.Deployment (status=pending)
  └── go Pipeline.Deploy(ctx, project, deployment, githubToken, onLog)
         Pipeline.Deploy:
           Semaphore.Acquire()             (blocks; max N concurrent)
           DB.UpdateDeploymentStatus(building)
           CloneRepo(buildDir, owner, repo, branch, token)
           GetHeadCommit(repoDir)          → update commit_sha/message
           projectconfig.Resolve(project)
           (RootDirectory shift)
           PatchNextConfig(projectDir)     (next.js only)
           DB.ListResolvedEnvVars + ListResolvedSecrets
           Encryptor.Decrypt each
           resolveDeploymentVariables      → (buildEnv, runtimeEnv)
           GenerateDockerfile              (or use user's)
           Docker.BuildImage               → streams to build_logs + WS
           DB.UpdateDeploymentStatus(deploying)
           Docker.ContainerExists(canonical)
           Docker.RunContainer(tempName, image, runtimeEnv, proxyNetwork)
           Docker.WaitForHealthy(60s)
           ensureEnvironmentDomains:
              For each domain in env:
                 ResolveVariantPlan(domain, env)
                 Manager.VerifyDomainDNS(each variant)
                 Manager.ProvisionDomain({PasswordProtected: project has pwd})
                 DB.UpdateDomainDNS/SSL
           if oldExists: Docker.StopContainer(old)
           Docker.cli.ContainerRename(new → canonical)
           DB.GetLiveDeployment → mark replaced
           DB.UpdateDeploymentContainer/Duration/Status(live)
           go (screenshot, 5s delay):
              find first domain.SSLStatus==active for env
              CaptureScreenshot → zenika/alpine-chrome → PNG
              DB.UpdateDeploymentScreenshot
  Semaphore.Release() via defer
```

## Full Pipeline Steps (numbered as emitted to user logs)

| # | Step | Code | On failure |
|---|------|------|-----------|
| 0 | "Waiting for build slot..." | `Semaphore.Acquire()` | — (blocks) |
| 1 | status=building, "Starting build for owner/repo@branch" | `DB.UpdateDeploymentStatus` | — |
| 2 | "Cloning repository..." → "Repository cloned" | `CloneRepo` | `fail("Clone failed")` |
| 2.5 | "Commit: {sha:8} {message}" | `GetHeadCommit` | log warning only |
| 3 | Resolve settings | `projectconfig.Resolve(project)` | `fail("Invalid build settings")` |
| 3.5 | RootDirectory shift | `filepath.Join(repoDir, RootDirectory)` + stat | `fail("Project root directory not found")` |
| 4 | "Patching Next.js config..." | `PatchNextConfig` (standalone runtime only) | log warning, proceed |
| 5 | Decrypt env vars + secrets | `Encryptor.Decrypt` per row | Skip + warn per-row |
| 5.5 | Split build/runtime | `resolveDeploymentVariables` | — |
| 6 | "Generating Dockerfile..." → "Dockerfile ready" | `GenerateDockerfile` | `fail("Dockerfile generation failed")` |
| 7 | "Building image {tag}..." | `Docker.BuildImage` (streams) | `fail("Docker build failed")` |
| 8 | status=deploying, "Deploying container..." | — | — |
| 9 | Check old canonical exists | `Docker.ContainerExists` | — |
| 10 | Start new w/ temp name | `Docker.RunContainer` | `fail("Container start failed")` |
| 11 | "Container started, waiting for health check..." | `Docker.WaitForHealthy(60s)` | Stop new; `fail("Health check failed")` |
| 12 | "Ensuring environment domains..." → per-domain "Provisioning" / "ready" | `ensureEnvironmentDomains` | Stop new; `fail("Domain provisioning failed")` |
| 13 | "Stopping old container..." (if oldExists) | `Docker.StopContainer(old)` | — |
| 14 | Rename new → canonical | `Docker.cli.ContainerRename` | — |
| 15 | Mark previous live `replaced` | `DB.UpdateDeploymentStatus(prev, "replaced")` | — |
| 16 | Finalize: container ID, duration, status=live, "Deployment live! ({d}s)" | `DB.UpdateDeploymentContainer/Duration/Status` | — |
| 17 | (async, 5s delay) screenshot | `CaptureScreenshot` | log only |

## `DockerfileData` Reference

```go
type DockerfileData struct {
    PackageManager  string   // "auto" | "bun" | "pnpm" | "npm" | "yarn"
    NodeVersion     string   // e.g. "22"
    InstallCommand  string   // e.g. "bun install"
    BuildCommand    string   // e.g. "bun run build"
    RootDirectory   string   // monorepo app path, relative to repo root
    OutputDirectory string   // runtime-specific: ".next" for standalone, "dist" for static
    Runtime         string   // projectconfig.RuntimeNextJSStandalone | RuntimeStatic
    BuildEnvVars    []db.ProjectVariable  // NEXT_PUBLIC_* only
}
```

`GenerateDockerfile(repoDir, data)` writes `Dockerfile.deployik` inside `repoDir` and returns its path. If a Dockerfile already exists at repo root, it is used as-is and the generator is skipped.

### `nextjs-standalone` stages

1. `deps` — install based on PackageManager
2. `builder` — copy repo, emit `ENV {KEY}="{value}"` for each BuildEnvVar (properly quoted), run `BuildCommand`
3. `runner` — minimal node:22-alpine, copy `.next/standalone` + `.next/static` + `public`, expose 3000, `CMD ["node", "server.js"]`

### `static` stages

1. `deps` — install based on PackageManager
2. `builder` — copy repo, run `BuildCommand` (which produces `OutputDirectory`)
3. `runner` — node:22-alpine with `npm i -g serve`, copy `OutputDirectory` to `/site`, `CMD ["serve", "-s", "/site", "-l", "3000"]`

### Package manager detection (`DetectPackageManager`)

Priority (first match wins):
1. `bun.lockb` or `bun.lock` → `bun`
2. `pnpm-lock.yaml` → `pnpm`
3. `yarn.lock` → `yarn`
4. `package-lock.json` → `npm`
5. InstallCommand starts with one of them → same
6. Fallback → `bun`

Lockfile search order: install directory (repo root if no RootDirectory, else RootDirectory) → app directory. Test covers monorepo scenarios (`dockerfile_test.go`).

## `resolveDeploymentVariables`

```go
func resolveDeploymentVariables(envVars, secretVars []db.ProjectVariable) (build, runtime []db.ProjectVariable) {
    for _, v := range envVars {
        if strings.HasPrefix(v.Key, "NEXT_PUBLIC_") {
            build = append(build, v)
        }
        runtime = append(runtime, v)
    }
    for _, s := range secretVars {
        runtime = append(runtime, s)
    }
    return
}
```

- `NEXT_PUBLIC_*` env vars go to BOTH build (baked in) and runtime (available to server code too).
- All non-public env vars → runtime only.
- All secrets → runtime only, never build-time.

## Screenshot Container

`zenika/alpine-chrome` with flags:
```
chromium-browser \
  --no-sandbox \
  --disable-gpu \
  --headless \
  --hide-scrollbars \
  --window-size=1280,800 \
  --screenshot=/out/{deploymentID}.png \
  https://{active_domain}
```

Bind-mount: `{ScreenshotDir}:/out`. Container runs on `proxy` network so it can resolve the deployed app container's name → useful for cross-container DNS probes though in practice we pass the public URL.

## Edge Cases

- **Concurrent deploys to the same project-env:** serialized by the global semaphore (default 1). If we raise the semaphore, per-project-env locking would need explicit handling — today it's accidentally safe via N=1.
- **Old canonical container not found:** blue-green still works; we just start new, rename, no stop step.
- **No domain configured for the environment:** `ensureEnvironmentDomains` emits "No domains configured for this environment" and returns nil — deployment still goes live on the container network, just no public URL.
- **DNS unverified custom domain:** emits "Skipping domain {d} until DNS is verified", marks SSL status `pending`, returns `ErrDNSNotVerified`. Deployment fails at the domain step.
- **Screenshot with no active SSL domain yet:** logs "Screenshot: no active domain for deployment" and returns — non-fatal.
- **User-provided Dockerfile at repo root:** generator is skipped entirely; user is responsible for their own monorepo + env handling.
