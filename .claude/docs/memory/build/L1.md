# Build Pipeline - Patterns & Rules
> Load when touching deployment flow, Dockerfile generation, or container lifecycle.
> Last validated: 2026-04-30

## Pipeline Shape (`internal/build/pipeline.go`)

Handlers create a `deployments` row with `status="queued"` and call `Pipeline.Dispatch(project, deployment, githubToken, onLog)`. `Dispatch` starts a pipeline-owned goroutine (unless `EnqueueOnly` is true in tests), derives the per-deploy context, catches panics, and then calls `Pipeline.Deploy(ctx, ...)`. `Deploy` always ends with a status transition to `live`, `failed`, or leaves `replaced` on the previous deployment.

Steps, in order:
1. **Semaphore acquire** тАФ global (default 1) via `build.NewSemaphore(n)` channel limiter. Blocks until a slot is free.
2. **Status тЖТ `building`**, emit `Starting build for owner/repo@branch`.
3. **Clone** тАФ `CloneRepo(buildDir, owner, repo, branch, token)` in `clone.go`. Shallow, OAuth-token auth. `defer os.RemoveAll(buildDir)` runs on exit.
4. **Capture HEAD commit** тЖТ update `commit_sha`, `commit_message`.
5. **Resolve settings** тАФ `projectconfig.Resolve(project)` merges framework preset + project overrides.
6. **Shift WORKDIR if RootDirectory set** тАФ monorepo support.
7. **Patch next.config** тАФ Next.js only. `PatchNextConfig(projectDir)` injects `output: 'standalone'` if not already present; idempotent.
8. **Resolve variables** тАФ `ListResolvedEnvVars` + `ListResolvedSecrets` (both already shared+scoped merged in SQL). Decrypt each with `p.Encryptor.Decrypt`.
9. **Split build vs runtime** тАФ `resolveDeploymentVariables(env, secrets)` in `variables.go` puts `NEXT_PUBLIC_*` env vars in the build set; everything else (including all secrets) in the runtime set.
10. **Generate Dockerfile** тАФ `GenerateDockerfile(repoDir, DockerfileData{...})` in `dockerfile.go`. Respects user-provided Dockerfile if present.
11. **Docker build** тАФ `p.Docker.BuildImage(ctx, repoDir, dockerfilePath, imageTag, onLine)`. Every line emitted to build_logs + WS hub.
12. **Status тЖТ `deploying`**, start container with **temp name** `{canonical}-{id8}`.
13. **Health check** тАФ `WaitForHealthy(ctx, containerID, 60s)`.
14. **Provision domains** тАФ `ensureEnvironmentDomains` iterates `domains WHERE environment = deployment.environment`; verifies DNS, then calls `domain.Manager.ProvisionDomain` (which re-runs certbot + rewrites nginx with password blocks if `project.PreviewPassword`/`ProductionPassword` set).
15. **Blue-green swap** тАФ stop old canonical container if present, then `ContainerRename(new, canonicalName)`.
16. **Mark previous live `replaced`**, new `live`.
17. **Screenshot (async, 5s delay)** тАФ find first SSL-active domain for this environment, run `zenika/alpine-chrome` in a temp container, write PNG to `{SCREENSHOT_DIR}/{deploymentID}.png`, update `deployments.screenshot_path`.

Container naming: `deployik-{project_name}-{environment}` (canonical). Temp name during deploy: `{canonical}-{deployID[:8]}`.

## Variable Resolution

| Set | Contains | Used as |
|-----|----------|---------|
| `buildEnvVars` | env-var `NEXT_PUBLIC_*` keys only | `ENV KEY="value"` lines in builder stage of Dockerfile |
| `runtimeEnvVars` | all env vars (non-public) + all secrets | Docker run `-e KEY=VALUE` via SDK |

**Rule:** `NEXT_PUBLIC_*` in the **secret** store is forbidden at the API layer (`envvars.go`). Enforced so secrets never get embedded into a public client bundle.

SharedтЖТscoped order: `shared` rows load first, then `preview`/`production` rows override by key (merge in SQL inside `queries_envvars.go`).

## Dockerfile Generation (`dockerfile.go`)

| Runtime | Trigger | Key stages |
|---------|---------|-----------|
| `nextjs-standalone` | framework `nextjs` (always) | node base тЖТ deps install тЖТ build тЖТ minimal runner copying `standalone` + `static` + `node server.js` |
| `static` | framework `vite`, `astro`, `static` | node base тЖТ deps install тЖТ build тЖТ `serve -s {output} -l 3000` |

**Package manager detection** (when `package_manager='auto'`): `bun.lockb`/`bun.lock` тЖТ `pnpm-lock.yaml` тЖТ `yarn.lock` тЖТ `package-lock.json` тЖТ command inference тЖТ fallback to bun. Lockfile searched first in the install directory (repo root for monorepos), then app dir.

**User-provided Dockerfile** at repo root wins тАФ skip generation, use as-is.

## Framework Defaults (`projectconfig.Resolve`)

| framework | runtime | default output | default build | default install |
|-----------|---------|---------------|---------------|-----------------|
| `nextjs` | `nextjs-standalone` | `.next` | `bun run build` | `bun install` |
| `vite` | `static` | `dist` | `bun run build` | `bun install` |
| `astro` | `static` | `dist` | `bun run build` | `bun install` |
| `static` | `static` | `dist` | `bun run build` | `bun install` |

Default Node version: `22`. The frontend `BuildSettingsFields` component mirrors these exact defaults тАФ keep them in sync.

## Do / Don't

| Do | Don't | Why |
|----|-------|-----|
| Emit progress via `emit(msg)` / `emitErr(msg)` (local closures) | `log.Printf` only | Logs must reach WS hub + `build_logs` table for the UI |
| Call `fail(err, msg)` for terminal errors | Return silently | Duration + status + error_message wiring is inside `fail` |
| `defer os.RemoveAll(buildDir)` on the clone temp | Leak `/tmp/deployik-builds/<id>` | Disk fills fast on busy VPS |
| Use temp container name for health check, rename only after healthy | Rename before healthy | Blue-green invariant: old container serves traffic until new passes |
| Keep async screenshot in its own goroutine | Block deploy completion on screenshot | Screenshot can take 10тАУ20s; user already saw `live` |

## Gotchas

- **`project_name` is used as both DNS subdomain AND container name suffix** тАФ slug validation `^[a-z0-9]([a-z0-9-]*[a-z0-9])?$` enforced in `projects.go`.
- **Docker SDK `cli.ContainerRename` is the swap mechanism** тАФ keep it direct; no intermediate state helps.
- **Screenshot only captures if there is a domain with `ssl_status='active'`** for that environment. New projects with only DNS-pending domains won't get screenshots until SSL provisions.
- **Semaphore is global per process.** Increasing `maxBuilds` in `main.go` must be matched by disk space + Docker concurrency headroom.
- **Next.js patcher handles both JS and TS configs**; it does a regex injection, not an AST rewrite. If the config uses unusual shapes (function wrappers), the patcher may no-op тАФ see tests in `nextjs_test.go` for supported shapes.
