# Frontend - Patterns & Rules
> Load when touching the React SPA (pages, hooks, data fetching, API client, auth hydration).
> Last validated: 2026-04-16

## Stack Snapshot

- React 19 + Vite 7 + TypeScript 5.8 (strict)
- TanStack Router (nested layouts), TanStack Query (staleTime 30s, retry 1)
- Zustand (auth + selected organization, `persist` for the latter)
- shadcn/ui (new-york variant, zinc dark theme), Tailwind 4 with `@tailwindcss/vite`
- Toaster: `sonner` (dark theme, bottom-right, richColors)
- Path alias: `@/` → `web/src/` (see `tsconfig.json` + `vite.config.ts`)

## Routing (`web/src/app/app.tsx`)

```
root
├── /login                         (public, redirects if authed)
├── /auth/callback                 (public, OAuth exchange)
└── protected (AppLayout, beforeLoad: hydrateAuthState)
    ├── workspace (WorkspaceLayout)
    │   └── /                      → Projects
    ├── /new                       → NewProject
    └── /projects/$id (ProjectLayout)
        ├── /                      → ProjectOverview
        ├── /deployments           → ProjectDeployments
        ├── /deployments/$did      → DeploymentDetail
        ├── /analytics             → ProjectAnalytics
        ├── /integration           → ProjectIntegration
        ├── /settings              → ProjectSettings (Build)
        ├── /settings/domains      → ProjectSettingsDomains
        ├── /settings/env          → ProjectSettingsEnv
        └── /settings/protection   → ProjectSettingsProtection
```

**Auth bootstrapping:** `hydrateAuthState()` (in `app.tsx`) calls `/api/auth/me` once per session load. It deduplicates concurrent calls via `authBootstrapPromise`. Every `beforeLoad` in a protected route awaits this before gating on `useAuthStore.getState().isAuthenticated`.

## Auth Store (`store/auth.ts`)

Stores **only** the `User` object and a `status: 'unknown' | 'authenticated' | 'unauthenticated'` flag. Never stores tokens. No `persist` — a page reload hits `hydrateAuthState()` again.

On 401 from the API client, auto-logout via `useAuthStore.getState().clearAuth()` and navigate to `/login`.

## API Client (`lib/api.ts`)

Class-based `ApiClient`:
- Base: uses relative `/api/*` in production, Vite proxy forwards to :8080 in dev.
- Every fetch: `credentials: 'include'` so HttpOnly cookies go out.
- 401 handling: one automatic refresh retry via `/api/auth/refresh`. On second 401, `clearAuth()` + redirect.
- Typed methods map to Go handler endpoints 1:1 (e.g. `api.listProjects({ organizationId })`, `api.getProjectDomains(id)`, `api.verifyDomain(projectId, domainId)`).
- Errors: throws `ApiError` with `{ status, code, message }` so callers can branch on `code` (e.g. `no_admin_access`).

## Shared Libs (`web/src/lib/`)

| File | Exports |
|------|---------|
| `api.ts` | `api` singleton, `ApiClient`, `ApiError` |
| `deployment-helpers.ts` | `DEPLOYMENT_STATUS_META`, `ENVIRONMENT_META`, `VARIABLE_SCOPE_META`, `isDomainReady`, `getEnvironmentDomains`, `getReadyEnvironmentDomains`, `getPrimaryEnvironmentUrl`, `formatRelativeDate`, `buildReleaseTagName`, `formatCompactNumber` |
| `utils.ts` | `cn()` (clsx + tailwind-merge) |

**Rule:** before adding a status badge / environment pill / variable scope badge anywhere, reach for `*_META` maps in `deployment-helpers.ts`. Do not redefine status labels or colors per page.

## Hooks (`web/src/hooks/`)

| Hook | Purpose |
|------|---------|
| `useBuildLogs(deploymentId)` | WebSocket subscription to `/ws/deployments/{did}/logs`, streams into a ring buffer for the viewer |
| `useDomainVerification(domainId, enabled)` | WebSocket `/ws/domains/{did}/logs` during verify + SSL |
| `use-mobile.ts` | shared breakpoint hook (shadcn sidebar needs it) |
| `use-organizations.ts` | React Query + Zustand bridge: list orgs + expose `selectedOrganizationId` + `setSelectedOrganizationId` |

## Types (`web/src/types/api.ts`)

Single file mirroring Go models: `User`, `Project`, `ProjectWithLatestDeployment`, `Deployment`, `Domain`, `AutoBuildConfig`, `ProtectionStatus`, `DeploymentListFilters`, `DeploymentListResponse`, `DomainLogEvent`, `VerifyDomainResponse`, etc. When adding a Go model field, update both places.

## Do / Don't

| Do | Don't | Why |
|----|-------|-----|
| Use `@/` imports | Deep relative paths (`../../../`) | Alias is in tsconfig + Vite |
| Import page components lazily if they get big | Eager-import everything in `app.tsx` | Initial bundle size — not yet a problem, but watch it |
| Use `useRouterState` / `useRouterLocation` from TanStack | `window.location` for routing decisions | TanStack already has nested context |
| Use shadcn variants (`variant="ghost" size="sm"`) | Arbitrary Tailwind overrides | Design consistency |
| Reuse `deployment-helpers` meta maps | Hardcode badge colors/labels | Already bit us during redesign |

## Gotchas

- **`status: 'unknown'` guard** — routes must await `hydrateAuthState()`. If you add a new protected route, copy the `beforeLoad` from existing ones; forgetting it sends users to `/login` even when authed.
- **Zustand `persist` on `store/organization.ts`** persists only the selected org id (string). Works fine across refreshes; clears on logout via `useOrganizationStore.getState().clearSelection()`.
- **Vite dev proxy** sends `/api/*` and `/ws/*` to `localhost:8080` (see `vite.config.ts`). Changing port on the Go side requires updating the proxy config.
- **`sonner` Toaster** is mounted once at the root route — import `toast` from `sonner` anywhere; no Provider needed.
- **Route params** use `$id`/`$did`; pass via `<Link to="/projects/$id" params={{ id }} />`. Typo-tolerant thanks to TanStack types.
