# Database - Patterns & Rules
> Load when touching SQL, models, migrations, or adding/modifying queries.
> Last validated: 2026-04-16

## Conventions

| Pattern | Rule | Example |
|---------|------|---------|
| IDs | Use `db.NewID()` (ULID) for primary keys; auto-increment `INTEGER` only for append-only tables (`build_logs`, `audit_logs`, `webhook_events`) | `project.ID = db.NewID()` |
| File layout | One `queries_<entity>.go` per aggregate, not one file per query. Keep cross-entity joins (`ListProjectsWithLatestDeployment`) with the primary entity. | `queries_projects.go`, `queries_deployments.go` |
| Models | Structs live in `models.go`. Derived/response DTOs (`ProjectWithLatestDeployment`, `DeploymentWithUser`) also in `models.go`. | — |
| Migrations | New SQL file `NNN_description.sql` in `internal/db/migrations/`. Embedded via `//go:embed`, applied in filename order, tracked in `_migrations` table. **Each migration runs in a transaction.** | `010_password_protection.sql` |
| Encrypted columns | Any TEXT column storing `user.github_token`, `projects.preview_password`, `projects.production_password`, `env_variables.value`, `auto_build_configs.webhook_secret` is AES-256-GCM ciphertext. | Decrypt via `h.Encryptor.Decrypt(value)` at read time. |
| Soft delete | `projects.status = 'deleted'` — rows stay. Domains/envvars/deployments use hard delete. | `DeleteProject` updates status only |
| JSON tags | `json:"-"` on ALL encrypted/secret fields so models never leak through the API. | `GithubToken string \`json:"-"\`` |

## Do / Don't

| Do | Don't | Why |
|----|-------|-----|
| Create new migration as `NNN_<slug>.sql` and let `Migrate()` apply it | Edit existing migration files | Drift between environments — `_migrations` won't re-run a rewritten file |
| Back up prod DB before migrating (`pnpm db:backup` on VPS) | `docker volume rm` / `docker compose down -v` on postgres/sqlite volumes | Permanent data loss — see global CLAUDE.md golden rules |
| Keep query functions pure (take `*DB` or `db.Querier`, return `(*Model, error)`) | Inline `database/sql` calls inside HTTP handlers | Handlers become untestable + mix concerns |
| Use `ListResolvedEnvVars` / `ListResolvedSecrets` in the pipeline | Roll your own shared→scoped merge | Merge logic must stay in one place |

## Key Query Signatures

```go
// Projects
database.ListProjectsWithLatestDeployment(userID, organizationID string) ([]ProjectWithLatestDeployment, error)
database.GetProjectForUser(projectID, userID string) (*Project, error) // owner OR org member
database.GetProject(projectID) (*Project, error) // admin-only path

// Deployments (with filtering + pagination)
database.ListDeploymentsFiltered(filter DeploymentFilter) (DeploymentListResponse, error)
database.GetLiveDeployment(projectID, environment) (*Deployment, error)

// Variables (shared + scoped merge)
database.ListResolvedEnvVars(projectID, environment) ([]ProjectVariable, error)
database.ListResolvedSecrets(projectID, environment) ([]ProjectVariable, error)
```

## Gotchas

- **SQLite driver** is `modernc.org/sqlite` — pure Go, no CGO. `go test ./...` works without build tags.
- **WAL mode** is enabled in `sqlite.go` Open(). Concurrent reads fine; a second writer will block.
- **`db.OpenMemory()`** gives each test an isolated in-memory DB — use in unit tests.
- **Nullable times** use `sql.NullTime` and serialize as `{"Time":..., "Valid":true/false}` if not custom-marshaled — existing JSON tags like `ssl_expires_at,omitempty` rely on this.
- **`GithubID` is int64** (GitHub uses 64-bit ids). Do not cast to int.
- **`queries_projects.go` is ~11k — find before edit**, especially for `GetProjectPassword` / `SetProjectPassword` / `ClearProjectPassword` which hit the encrypted columns.
