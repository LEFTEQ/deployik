## Architecture

```
modernc.org/sqlite (pure Go, no CGO)
  └─ db.Open(path) → sets PRAGMAs: journal_mode=WAL, synchronous=NORMAL, foreign_keys=ON, busy_timeout=5000
     ▸ Migrate() runs migrations/NNN_*.sql in order (embedded via //go:embed)
     ▸ each migration wrapped in a transaction, recorded in _migrations table
     ▸ queries live in queries_<entity>.go files — all methods on *DB wrapper
```

## Query Contracts (selected)

### ListResolvedEnvVars / ListResolvedSecrets
```go
func (d *DB) ListResolvedEnvVars(projectID, environment string) ([]ProjectVariable, error)
```
Returns merged shared + scoped rows. SQL union + GROUP BY key with priority to scoped rows. Values are **still encrypted** — caller decrypts.

### ListProjectsWithLatestDeployment
```go
func (d *DB) ListProjectsWithLatestDeployment(userID, organizationID string) ([]ProjectWithLatestDeployment, error)
```
`organizationID == ""` → all workspaces the user can access. Joins `projects` with the most recent deployment per project (LEFT JOIN + window function / correlated subquery).

### GetProjectForUser
```go
func (d *DB) GetProjectForUser(projectID, userID string) (*Project, error)
```
Returns nil (no error) if the user cannot see the project. Matches on `user_id = userID` OR `organization_id IN (memberships of userID)`.

### ListDeploymentsFiltered
```go
func (d *DB) ListDeploymentsFiltered(filter DeploymentFilter) (DeploymentListResponse, error)
```
Dynamic WHERE from filter fields (branch, environment, status, triggered_by, from, to). Joins `users` for `Username` + `AvatarURL`. Respects `Limit`/`Offset` for pagination.

### SetProjectPassword / ClearProjectPassword
```go
func (d *DB) SetProjectPassword(projectID, environment, encryptedValue string) error
func (d *DB) ClearProjectPassword(projectID, environment string) error
```
Writes to `preview_password` or `production_password` column based on environment. Invalid environment → error.

## Edge Cases

- **Migrations in a transaction:** if migration N fails midway, the whole N is rolled back; `_migrations` is NOT updated. On retry it re-runs from scratch. CREATE TABLE IF NOT EXISTS is still preferred to be safe.
- **`json:"-"` protection:** ALL secret columns (github_token, passwords, webhook_secret, token_hash) have `json:"-"` to avoid accidental exposure in any API response.
- **SoftDelete on projects + ON DELETE CASCADE:** because projects use soft-delete, dependent rows (deployments, domains, env_variables, auto_build_configs) also stay. When hard-deleting a project for cleanup, CASCADE triggers remove them. Keep soft-delete for normal flow.
- **WAL mode on SQLite:** allows concurrent readers + one writer. Writer serialization is fine for our throughput; backups need to `VACUUM INTO` (see `internal/backup/sqlite.go`), not a simple file copy.
- **ULID over UUID:** ULIDs sort lexicographically by time → list queries with `ORDER BY id DESC` naturally return newest-first.

## Migration Notes

- **Migration numbering is gap-free.** If you delete a migration, ALL environments must re-bootstrap — don't do it.
- **Adding a column:** ALTER TABLE in a new migration. Populate defaults directly in the ALTER or in a follow-up DML step inside the same migration file.
- **Renaming a column:** Avoid. Prefer adding a new column + migrating data + dropping the old one in a later migration after code stops reading it.
- **Changing a constraint:** SQLite doesn't support ALTER TABLE ADD CONSTRAINT. Workaround pattern: new table + INSERT ... SELECT + DROP old + RENAME new. Do this in one migration transaction.
