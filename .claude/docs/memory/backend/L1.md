# Backend - Patterns & Rules
> Load when touching HTTP handlers, routing, middleware, or cross-cutting server concerns.
> Last validated: 2026-04-16

## Conventions

| Pattern | Rule | Example |
|---------|------|---------|
| Handler struct | Each handler is a struct with DB/Encryptor/Audit deps injected in `router.go` | `&handlers.ProjectHandler{DB, Docker, Manager, Encryptor, Audit, Analytics, DevMode}` |
| Method receivers | Handlers expose methods that implement `http.HandlerFunc` signature | `func (h *ProjectHandler) List(w, r)` |
| Response writer | Use `writeJSON(w, status, payload)` from `handlers/helpers.go` — do NOT call `json.NewEncoder(w).Encode` directly |
| Error shape | `{"error": "<message>"}` with correct HTTP status; for codes with actionable UX include `error` + `message` fields (autobuild 403 pattern) |
| URL params | `chi.URLParam(r, "id")` / `"did"` / `"key"` — never parse the raw path |
| Query params | `r.URL.Query().Get(...)` then `strings.TrimSpace` before use |
| Claims extraction | `claims := auth.GetClaims(r.Context())` — assumes `middleware.Authenticate` ran |
| Rate limiting | Mutation routes wrap with `mutationLimiter.Middleware("action_name")`; auth uses `oauthLimiter`/`refreshLimiter`; WS uses `wsLimiter` |

## Do / Don't

| Do | Don't | Why |
|----|-------|-----|
| Call `loadAuthorizedProject(w, r, h.DB, id)` before acting on a project | Call `h.DB.GetProject(id)` directly in a handler | Bypasses ownership + admin check, leaks cross-tenant data |
| Record sensitive mutations via `h.Audit.Record(audit.Entry{...})` | Silently mutate without audit | Security reviews rely on `audit_logs` |
| Return early with `writeJSON` on error | Fall through to success path after error | Tests will still see 200 |
| Wrap webhook errors with HTTP status codes | Panic on bad signatures | Recovery middleware catches panics but wastes a stack unwind |
| Add new handler deps to `RouterConfig` struct | Use package-level globals | `main.go` is the single wiring point |

## Middleware Chain (in order)

```
chi.Logger → chi.Recoverer → chi.RequestID → chi.RealIP → middleware.CORS
  → (per-route) rate limiter → middleware.Authenticate (protected group only)
```

`middleware.CORS` is allowlist-driven via `Config.AllowedOrigins`. `middleware.Authenticate` checks `Authorization: Bearer <jwt>` header first, then falls back to the `deployik_access_token` cookie.

## Route Groups

Router structure in `internal/api/router.go`:

```
/api/site-auth/verify        Public (nginx calls this)
/api/site-auth/check         Public (nginx auth_request)
/api/health                  Public
/api/auth/github, callback   Public (oauthLimiter)
/api/auth/dev-login          Public (DEV_MODE only)
/api/auth/refresh, logout    Public (refreshLimiter)
/api/webhooks/github         Public (signature-validated)
/api/auth/me                 Protected
/api/github/*                Protected
/api/organizations           Protected
/api/platform                Protected
/api/projects/*              Protected (all mutations are rate-limited)
/ws/deployments/{did}/logs   WebSocket (cookie OR ?token= OR header)
/ws/domains/{did}/logs       WebSocket
(any other path)             SPAHandler fallback
```

## Authorization Helpers (`handlers/access.go`)

```go
project, claims, ok := loadAuthorizedProject(w, r, h.DB, projectID)
if !ok { return } // helper already wrote 404/500 and logged
```

```go
deployment, claims, ok := loadAuthorizedDeployment(w, r, h.DB, deploymentID)
if !ok { return }
```

Both delegate to `authz.LoadProject` / `authz.LoadDeployment`:
- Admin role (claims.Role == "admin") → bypass (get by ID only)
- Regular user → `GetProjectForUser(id, userID)` which joins through ownership **and** org membership

## Helper Utilities

| Helper | Location | Purpose |
|--------|----------|---------|
| `writeJSON(w, status, v)` | `handlers/helpers.go` | Sets `Content-Type: application/json` + encodes |
| `auth.GetClaims(ctx)` | `internal/auth/context.go` | Returns `*auth.Claims` or nil |
| `audit.Recorder.Record(entry)` | `internal/audit/recorder.go` | Inserts into `audit_logs` — never blocks |

## Gotchas

- **Rate limiter is in-memory per-IP** — does not survive restarts, does not sync across replicas. Single-instance deploy only.
- **The handler struct `DevMode` bool controls mock data** in `ProjectHandler.ListGithubRepos` / `ListGithubBranches`. Production must have `DEV_MODE` unset.
- **`/api/site-auth/*` is registered OUTSIDE the `/api` chi.Route group** because nginx calls it without auth cookies — do not add auth middleware there.
- **Webhook route bypasses JWT auth** entirely; it validates HMAC against the stored per-project webhook secret instead.
- **Panic recovery is `chi.Recoverer`** — handlers must not panic on user input; return `writeJSON` with the appropriate status.
