## Architecture

```
main.go
  └─ loads config → encryptor → db → wsHub → domainManager → analyticsService → pipeline
     → api.NewRouter(RouterConfig{...})
         └─ chi.Mux
              Logger → Recoverer → RequestID → RealIP → CORS
              ├─ /api/site-auth/{verify,check}    (public, called by nginx)
              ├─ /api (sub-router)
              │    ├─ /health
              │    ├─ /auth/github, /auth/github/callback    (oauthLimiter)
              │    ├─ /auth/dev-login                        (DEV_MODE only)
              │    ├─ /auth/refresh, /auth/logout            (refreshLimiter)
              │    ├─ /webhooks/github                       (webhookLimiter)
              │    └─ Group(Authenticate)
              │         /auth/me, /organizations, /platform
              │         /github/{repos,branches}
              │         /projects/*        (mutationLimiter on POST/PUT/PATCH/DELETE)
              │         /deployments/{did}/{logs,screenshot}
              │         /site-auth/*       (wait — no, those are public above)
              └─ /ws/deployments/{did}/logs, /ws/domains/{did}/logs   (wsLimiter)
              NotFound → SPAHandler   (serves embedded web_dist)
```

## Handler Struct Pattern

Every handler is a struct with explicit dependencies; no package-level state. Wiring happens in `router.go`:

```go
projectHandler := &handlers.ProjectHandler{
    DB:        cfg.DB,
    Docker:    dockerClient,
    Manager:   cfg.DomainManager,
    Encryptor: cfg.Encryptor,
    Audit:     auditRecorder,
    Analytics: cfg.Analytics,
    DevMode:   cfg.DevMode,
}
r.Get("/projects", projectHandler.List)
```

Benefits:
- No init-order pitfalls.
- Tests construct handlers with in-memory DB + nil Docker cleanly.
- Adding a dep = adding a struct field + setting it in `router.go`.

## Authorization Pattern (`handlers/access.go`)

```go
func loadAuthorizedProject(w http.ResponseWriter, r *http.Request, database *db.DB, projectID string) (*db.Project, *auth.Claims, bool) {
    claims := auth.GetClaims(r.Context())
    if claims == nil {
        writeJSON(w, http.StatusUnauthorized, map[string]string{"error": "unauthorized"})
        return nil, nil, false
    }
    project, err := authz.LoadProject(database, claims, projectID)
    if err != nil {
        writeJSON(w, http.StatusInternalServerError, map[string]string{"error": "failed to load project"})
        return nil, nil, false
    }
    if project == nil {
        writeJSON(w, http.StatusNotFound, map[string]string{"error": "project not found"})
        return nil, nil, false
    }
    return project, claims, true
}
```

`authz.LoadProject` dispatches:
- `claims.Role == "admin"` → `db.GetProject(id)` (no ownership check).
- Otherwise → `db.GetProjectForUser(id, claims.UserID)` which returns nil if the user is neither the creator nor a member of the project's organization.

The helper returns `(nil, false)` so the caller can just `if !ok { return }`.

## Rate Limiter Groups

Instantiated in `router.go`:
- `oauthLimiter` — 20 req/min/IP (auth endpoints)
- `refreshLimiter` — 30 req/min/IP (refresh, logout)
- `mutationLimiter` — 60 req/min/IP (all project mutations)
- `wsLimiter` — 30 req/min/IP (WS upgrades)
- `webhookLimiter` — 60 req/min/IP (GitHub webhook POST)

Limiter keys include an action name so you can grep `audit_logs` for correlated blocks.

## Audit Log Entries

`audit.Recorder.Record(audit.Entry{...})` inserts into `audit_logs`. Entries are fire-and-forget (logged on error, never block the request). Conventions:

|--------|--------------|--------------|
| `auth.login` | `user` | AuthHandler.GithubCallback |
| `auth.refresh` | `session` | AuthHandler.RefreshToken |
| `auth.logout` | `session` | AuthHandler.Logout |
| `project.create` / `.update` / `.delete` | `project` | ProjectHandler |
| `deployment.trigger` | `deployment` | DeploymentHandler |
| `autobuild.update` / `.delete` | `auto_build_config` | AutoBuildHandler |
| `protection.update` / `.regenerate` | `project` | ProtectionHandler |
| `domain.add` / `.delete` / `.verify` | `domain` | DomainHandler |

## Edge Cases

- **Empty `AllowedOrigins`:** `CORS` middleware treats empty as "no origins allowed" except for same-origin requests. In dev `http://localhost:5173` must be explicitly listed.
- **Missing `X-Hub-Signature-256` on webhook:** treated as invalid signature → 401. We do NOT accept unsigned webhooks even if the secret is empty.
- **Bearer + cookie both present:** Bearer wins (header-first precedence in `Authenticate`).
- **Admin accessing another user's personal org:** `LoadProject` returns the project; `ListProjects` with `organization_id=` of another user returns only projects the admin can see — admins DO see everything today.
- **DEV_MODE with partial config:** `main.go` falls back to hardcoded dev values for JWT/ENCRYPTION keys — anything encrypted in that session is throwaway.
