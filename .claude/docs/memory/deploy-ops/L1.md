# Deploy-Ops (Docker, CI/CD, Scripts) - Patterns & Rules
> Load when touching the Docker image, compose file, GitHub Actions workflow, or anything under `scripts/`.
> Last validated: 2026-04-16

## Docker Image (`docker/Dockerfile`)

Multi-stage, built by `make docker-build`:

| Stage | Base | Produces |
|-------|------|---------|
| 1 — web | `oven/bun` | `web/dist` (Vite build) |
| 2 — server | `golang:1.25-alpine` | Go binary with `//go:embed web_dist` (copied from stage 1 into `cmd/server/web_dist`) |
| 3 — runtime | `alpine:latest` | Minimal image: `ca-certificates`, `git`, `docker-cli`, the binary |

The runtime needs `docker-cli` because the running Deployik container SHELLS OUT to the host docker daemon via bind-mounted `/var/run/docker.sock` to manage app containers.

## docker-compose (`docker/docker-compose.yml`)

Key mounts/config:
- `/var/run/docker.sock:/var/run/docker.sock` — so Deployik can start/stop app containers
- `/opt/nginx-proxy/conf.d:/opt/nginx-proxy/conf.d` — writes nginx configs (alongside the `nginx-proxy` container)
- Named volume for SQLite data (`/app/data`)
- Joined to external `proxy` network where deployed apps and the nginx-proxy container live

## CI/CD (`.github/workflows/ci.yml`)

Triggers on push/PR to `main`. Three jobs:

| Job | On | Does |
|-----|----|------|
| `test` | any | Go tests + `cd web && bun install && bunx tsc --noEmit && bun run build` |
| `build-and-push` | push to main | Multi-stage Docker build → push to `ghcr.io/lefteq/lovinka-deployik` with SHA tag + `latest` |
| `deploy-vps` | push to main | SSH to VPS, `docker compose pull app`, `docker compose up -d`, health check |

## Scripts (`scripts/`)

| Script | Purpose |
|--------|---------|
| `seed-dev.sh` | Resets local dev DB, creates a dev user + a couple of projects via the dev-login endpoint. Invoked by `make dev-seed`. |
| `deploy.sh [tag]` | Manual deploy: builds image (or pulls `tag`), scp's compose file, SSHes to `deploy@203.0.113.10`, `docker compose up -d`. |
| `deploy-vps.sh` | Companion helper run ON the VPS by deploy.sh — pulls the image and restarts. |

## `cmd/backup/main.go`

Standalone binary (built separately) that takes an online SQLite snapshot using `VACUUM INTO` plus `PRAGMA integrity_check`. Intended to be invoked from an infra-managed systemd timer or external cron. Output is a consistent, point-in-time `.db` file safe to ship off-site. Core logic lives in `internal/backup/sqlite.go` (`CreateSQLiteSnapshot`).

## Environment Config (`internal/config/config.go`)

Required: `JWT_SECRET`, `ENCRYPTION_KEY`, `GITHUB_CLIENT_ID`, `GITHUB_CLIENT_SECRET`. Missing any one of these aborts startup — UNLESS `DEV_MODE=true`, in which case dev fallbacks are logged and applied.

See root `CLAUDE.md` → "Environment Variables Reference" for the full optional list. Most notable optional ones:
- `ALLOWED_GITHUB_USERS` — login allowlist (empty = allow all).
- `ADMIN_GITHUB_USERS` — explicit admin list. If empty, first signup auto-promotes.
- `WEBHOOK_URL` — public URL GitHub posts to; defaults to `{FRONTEND_URL}/api/webhooks/github`.
- `NGINX_CONF_DIR`, `PROXY_*` — bind-mount paths for the nginx/certbot integration.

## Do / Don't

| Do | Don't | Why |
|----|-------|-----|
| Edit infra in `~/Documents/Work/infra-repo/` locally, then `scp` to VPS | Edit files directly on the VPS | Config drift (global CLAUDE.md rule) |
| `docker compose restart <service>` | `docker compose down -v` / `docker volume rm <postgres_vol>` | `-v` destroys volumes — data loss |
| Run `pnpm db:backup` in lovinka project before migrations | Hope the migration is idempotent | Restore path matters; have the backup |
| Tag releases through the in-app release dialog (creates git tag on main) | Manually tag in GitHub UI | Keeps `deployments.commit_sha` ↔ tag coupled |

## Gotchas

- **The Go binary embeds `cmd/server/web_dist`** via `//go:embed all:web_dist`. That directory is populated by `make build` (copies from `web/dist`) OR the Docker build (stage 1 output). If you `go build ./cmd/server/` **without** running `make build-web` first, the binary serves a broken SPA.
- **Docker image does not include a healthcheck for the app container itself** — nginx-proxy trusts container_name resolution + the Go server's HTTP 200 on `/api/health`.
- **Tests use `db.OpenMemory()`** (pure-Go modernc SQLite) — no external dependency. `go test ./...` works anywhere.
- **`cmd/server/web_dist` is gitignored** — committing it bloats the repo.
- **VPS deploy key** is configured per-repo; the SSH private key lives as a GitHub Actions secret. No user action needed per deploy.
